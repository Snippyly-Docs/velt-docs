## VeltRecorderTool

<Steps>
<Step title="Recording Types">

Sets the `VeltRecorderTool` recording mode.

`Default: 'audio'`

The `VeltRecorderTool` can be configured to record a user's audio, video or screen.

To set the configuration mode, set the `type` attribute to one of the following:
 - all
 - audio 
 - video
 - screen

```js
<VeltRecorderTool type='all' />
<VeltRecorderTool type='audio' />
<VeltRecorderTool type='video' />
<VeltRecorderTool type='screen' />
```

</Step>

<Step title="Recorder Panel Id">

Sets the `panelId` that the Recording will be played on.

You must also set the `VeltRecorderPlayer` to the same `panelId` for the recording to show up on it.

```js
<VeltRecorderTool panelId="YOUR_PANEL_ID" />
<VeltRecorderPlayer panelId="YOUR_PANEL_ID"/>
```

</Step>

<Step title="Button Label">

Sets a custom label for the `VeltRecorderTool`.

```js
<VeltRecorderTool buttonLabel="Your Label Text" />
```

</Step>

</Steps>


## VeltControlPanel

<Steps>

<Step title="Floating Mode vs. Thread Mode">

The `VeltRecorderControlPanel` can be configured in either `Floating` mode or `Thread` mode. 

<Frame>
![](https://files.readme.io/38580c5-Frame_8253.png)
</Frame>

Floating mode will show a preview in the bottom left of the page, no matter where you put the component.

Thread mode will show the component wherever it is placed.

To configure the mode, set the `mode` attribute to `floating` or `thread`:

```js
<VeltRecorderControlPanel mode="floating" />
<VeltRecorderControlPanel mode="thread" />
```


</Step>

<Step title="onRecordedData callback">

Whenever a recording is created, the `VeltRecorderControlPanel` component emits a callback event called `onRecordedData` with data from the `VeltRecorderPlayer` that was used to create the recording.. 

This is the response object that you get back:

| Field    | Type                          | Example                                                                            |
| -------- | ------------------------------| -----------------------------------------------------------------------------------|
| id       | string                        | "recorder_id"                                                                      |
| tag       | string                        | `<snippyly-recorder-player recorder-id=\"RECORDER_ID\"></snippyly-recorder-player>` |

```js
import { VeltRecorderControlPanel } from '@veltdev/react'

const SomeComponent = () => {

    const onRecordedData = (id,tag) => {
        console.log(id,tag)
    }
    return (
    	//... rest of the component code...
      <VeltRecorderControlPanel mode="thread" onRecordedData={onRecordedData} />

      //... rest of the component code...
    )
}

export default SomeComponent;

```
</Step>

</Steps>

## VeltRecorderPlayer


<Steps>


<Step title="Show Transcript">
Whether a transcript of a recording gets shown or not.

`Default: true`

The `VeltRecorderPlayer` will show a transcript of an audio, video or screen recording.

To enable or disable the transcript, set the `show` attribute:

```jsx
<VeltRecorderPlayer show={true}/>
```


</Step>


<Step title="onDelete callback">
The creator of the recordings will see an option to delete the recording after it has been recorded. Once the user deletes the recording, the `VeltRecorderPlayer` component will emit an event called `onDelete` with the recorder ID. You can listen to the callback and use the response object to perform any clean up on your end. 

This is the response object that you get back:

| Field    | Type                          | Example       |
| -------- | ------------------------------| --------------|
| id       | string                        | "recorder_id" |

```js
import { VeltRecorderPlayer } from '@veltdev/react'

const SomeComponent = () => {
  
  const onDelete = (data) => {
    console.log('on delete: ', data);
  }
   
  return (
  	//... rest of the component code...
		<VeltRecorderPlayer recorderId={RECORDER_ID} onDelete={(data) => onDelete(data)} />

    //... rest of the component code...
  )
}

export default SomeComponent;
```

</Step>


</Steps>
